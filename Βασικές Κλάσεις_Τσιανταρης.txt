TΣΙΑΝΤΑΡΗΣ ΔΗΜΗΤΡΗΣ- ΒΑΣΙΚΕΣ ΚΛΑΣΕΙΣ

1.Account(type, username, passwd, phone, email): Κλάση για τους λογαριασμούς που θα υπάροχυν(και θα μπορούν να δημιουργηθούν στο σύστημα). 
Το type έχει επιλογές admin ή client και η κάθε επιλογή δίνει τα αντίστοιχα δικαίωμα στον χρήστη.Username και passwd αφορούν τα
credentials του χρήστη. Αν ο χρήστης είναι client θα υπάρχουν και κατοχυρωμένες διευθύνσεις για τις παραγγελίες του(τουλάχιστον μια).

2.StoreType(storeType): Το storeType θα βοηθούσε να είναι ξεχωριστή κλάση που θα υπάρχει σαν παράμετρος στην κλάση Store(3) έτσι
ώστε να είναι πιο εύκολη η αναζήτηση ανά κατηγορίες.

3.Store(storeName, storeCode, storeType, geolocation, operatingHours, products): Το storeName θα είναι το όνομα του μαγαζιού. Το
storeCode θα είναι ένας integer κωδικός που θα ξεχωρίζει το κατάστημα από τα υπόλοιπα (και θα συνδέεται με μεγαλύτερη ευκολία στις
υπόλοιπες μεταβλητές. Το storeType όπως εξηγείται στο (2) θα είναι ο τύπος καταστήματος. Το geolocation θα είναι, ανάλογα με το API
που θα χρησιμοποιήσουμε μία μεταβλήτη που θα 'κοιτάει' στην τοπογραφική διεύθυνση του καταστήματος. Το operatingHours θα είναι οι 
ώρες λειτουργίας του καταστήματος. To products(5) θα είναι τα διαθέσιμα προιόντα που θα υπάρχουν για το κατάστημα.

4.Geolocation(geoCode, ??): Θα μελετηθεί εκ νέου ανάλογα με το API. Λογικά θα δημιουργηθούν και άλλες κλάσεις τύπου tracking etc.

5.Product(name, productCode, type, price): Θα περιγράφει ένα προϊόν(name) και θα έχει έναν μοναδικό κωδικό(integer) που θα συνδέει το 
προϊόν με την βάση δεδομένων. To type θα αναφέρεται στο αν είναι φαγητό, ποτό κλπ. Το price θα είναι η αντίστοιχη τιμή. Πρέπει να βρεθεί
ένας τρόπος για να ξεχωρίζουν οι τιμές για το κάθε κατάστημα ΞΕΧΩΡΙΣΤΑ σε περίπτωση που έχουν προϊόν με ίδια περιγραφή(εδώ θα βοηθήσει
το productCode).

6.Order(orderID, date, store, products, cost, paymentType, comments): Η κλάση Order θα περιγράφει μια παραγγελία. Θα υπάρχει timestamp τύπου Date,
από ποιο κατάστημα, περιγραφή προϊόντων, συνολικό κόστος, τρόπος πληρωμής και σχόλια. Θα γίνεται implement(ή θα συνδέεται άμεσα με κάποιο
μηχανισμό) με μία κλάση Cart(7).

7.Cart(order): Θα κάνει implement την Order και θα δέχεται παραγγελίες.Θα μελετηθεί περαιτέρω σε επόμενες συναντήσεις.

8.Payment(paymentType, sum): Θα καταγράφει το συνολικό ποσό(sum) με το οποίο ολοκληρώνεται η παραγγελία και αναλόγως με τον επιλεγμένο 
τρόπο πληρωμής ίσως προχωράει σε περαιτέρω ενέργειες για την άμεση εξόφληση(επιλογή Online πληρωμής)**.
	**Πρέπει να βρεθεί ένας τρόπος να αποθηκεύονται στοιχεία πληρωμής(ίσως στην κλάση Account).Θα μελετηθεί σε συνάντηση
